**code** — it does not grant rights to the project's **name or logo**.

## You may, without asking

- Use the name "onepool" to truthfully refer to this project or unmodified builds of it

- Distribute unmodified copies under the name "onepool".
- Fork the code under the AGPL and develop it under a **different name**.

## You may not, without written permission

- Distribute a **modified** version under the name "onepool" or a confusingly similar
  name (e.g. "onepool-pro", "OnePoolX").
- Use the name or logo in a way that implies endorsement by, or origin from, the
  onepool project.
- Register domains, package names, or social accounts using the mark in a way likely
  to confuse users about their official status.

## Why this exists

The AGPL keeps the code open. This policy keeps the name meaningful: when someone
installs "onepool", they should get this project — not a rebranded, altered, or
proprietary derivative. This is the standard pairing used by projects like Firefox
and Debian: open code, protected identity.

Questions or permission requests: open an issue on the project repository.
